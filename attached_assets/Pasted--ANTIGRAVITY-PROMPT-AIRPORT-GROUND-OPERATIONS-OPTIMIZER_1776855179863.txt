📦 Project: Airport Ground Operations Optimizer Using AI

🔹 Problem Summary
Airports suffer from poor ground resource coordination, unpredictable delays, and inefficient scheduling. This system uses predictive AI + real‑time optimisation to solve that.

🧠 Machine Learning Component (model.pkl)

Training script: src/train_model.py
- Generate synthetic dataset with 50,000+ samples using realistic distributions.
- Features (must be stored as feature_columns.json):
  - hour (0‑23)
  - day_of_week (0‑6)
  - flight_type (Domestic/International/Cargo – one‑hot encoded)
  - num_ground_crew (3‑12)
  - gate_distance (50‑800 metres)
  - weather_impact (0 = clear, 1 = rain, 2 = fog, 3 = storm)
  - congestion_level (0‑100)
  - scheduled_arrival_delay (minutes, -15 to +60)
  - baggage_volume (0‑500 bags)
  - stand_type (nose‑in, remote, tow‑in)
  - previous_flight_delay (minutes)
- Target variable: actual_turnaround_time (minutes, 25‑120).
- Model: XGBoost Regressor (or RandomForest with hyperparameter tuning).
- Output files saved in models/:
  - model.pkl
  - feature_columns.json
  - scaler.pkl
  - label_encoders.pkl (for categoricals)
- Evaluation: Print RMSE, MAE, R². Plot feature importance.

Inference module: src/predictor.py
- Load model.pkl, scaler, encoders.
- Function predict_turnaround(flight_data_dict) → returns predicted minutes + bottleneck probability.
- Function predict_bottleneck_resource(flight_id) → returns which resource (crew/baggage/gate/tug) is likely to cause delay.

🏗️ Backend & Real‑time Data (FastAPI + WebSockets)

backend/main.py
- REST endpoints:
  - GET /api/flights/current → list of all active flights with live status.
  - GET /api/resources → utilisation of gates, tugs, boarding bridges, cleaning crews, baggage belts.
  - POST /api/optimize → runs optimisation algorithm (see below) and returns new schedule.
  - GET /api/predict/{flight_id} → return AI prediction.
- WebSocket endpoint: /ws/live – broadcasts every 3 seconds:
  - Updated flight positions (taxi, at gate, boarding, pushback).
  - New predictions.
  - Bottleneck alerts.
- Background task: mock_data_stream() updates flight states, randomly introduces delays, and logs events to SQLite.

Database: airport_ops.db (SQLite with SQLAlchemy)
Tables:
- flights (id, flight_number, airline, gate, scheduled_arrival, actual_arrival, status, predicted_turnaround, actual_turnaround, bottleneck_flag)
- resources (id, type, current_assignment, utilisation_percent, status)
- events (timestamp, flight_id, event_type, description)

---

🧠 Optimisation Algorithm (src/optimizer.py)
- Objective: Minimise total turnaround delay across all active flights.
- Constraints: gate availability, crew shift limits, equipment capacity.
- Method: Greedy heuristic with look‑ahead (re‑assign resources every 30 seconds based on AI predictions).
- Must expose function reassign_resources(current_flights, predictions) that returns new assignments and logs changes.
